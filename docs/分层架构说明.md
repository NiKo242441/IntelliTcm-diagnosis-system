# 企业级分层架构说明

## 架构概览

本项目采用**四层架构**（Presentation → Application → Domain → Infrastructure），符合DDD（领域驱动设计）思想，适用于企业级SSR应用。

```
┌─────────────────────────────────────────────────────────────┐
│                    表现层 (Presentation)                     │
│         Razor Pages / Controllers / View Models             │
│                      ↓ 调用 ↓                               │
├─────────────────────────────────────────────────────────────┤
│                    应用层 (Application)                      │
│         应用服务 / DTO映射 / 业务流程编排                     │
│                      ↓ 调用 ↓                               │
├─────────────────────────────────────────────────────────────┤
│                    领域层 (Domain)                           │
│         领域模型 / 领域服务 / 仓储接口                        │
│                      ↓ 依赖 ↓                               │
├─────────────────────────────────────────────────────────────┤
│                    基础设施层 (Infrastructure)               │
│         EF Core / Redis / 外部API / 文件系统                 │
└─────────────────────────────────────────────────────────────┘
```

## 各层职责说明

### 1. 表现层 (Presentation Layer) - `TcmAiDiagnosis.Web`

**职责**: 处理HTTP请求，返回HTML页面或JSON数据

**包含内容**:
```
TcmAiDiagnosis.Web/
├── Areas/                      # 按角色划分的Razor Pages
│   ├── Doctor/                 # 医生端页面
│   │   └── Pages/
│   ├── Patient/                # 患者端页面（待开发）
│   ├── Pharmacist/             # 药剂师端页面
│   └── Manager/                # 管理员端页面（待开发）
├── Pages/                      # 公共页面
├── Controller/                 # API控制器（返回JSON）
├── ViewModels/                 # 视图模型（View特有）
├── Views/                      # 共享视图
├── Middleware/                  # HTTP中间件
├── Attributes/                 # 自定义特性
├── Extensions/                 # 扩展方法
└── wwwroot/                    # 静态资源
```

**职责边界**:
- ✅ 接收用户输入，验证请求格式
- ✅ 调用应用层服务处理业务
- ✅ 将结果转换为视图模型
- ✅ 渲染HTML页面或返回JSON
- ❌ 不包含业务逻辑
- ❌ 不直接访问数据库

**规范**:
- Razor Pages使用`@page`指令定义路由
- 使用`[RequirePermission]`特性控制访问权限
- Controller使用`[ApiController]`特性
- 所有页面必须有对应的ViewModel

### 2. 应用层 (Application Layer) - `TcmAiDiagnosis.Application` (待创建)

**职责**: 编排业务流程，协调领域服务，处理事务

**包含内容**:
```
TcmAiDiagnosis.Application/
├── Services/                   # 应用服务
│   ├── PatientAppService.cs    # 患者应用服务
│   ├── VisitAppService.cs      # 就诊应用服务
│   ├── TreatmentAppService.cs  # 治疗方案应用服务
│   └── ...
├── DTOs/                       # 数据传输对象
│   ├── Patients/               # 患者相关DTO
│   ├── Visits/                 # 就诊相关DTO
│   └── Treatments/             # 治疗相关DTO
├── Mappings/                   # AutoMapper配置
│   ├── PatientMapping.cs
│   └── ...
├── Interfaces/                 # 应用服务接口
│   ├── IPatientAppService.cs
│   └── ...
└── Validators/                 # 输入验证（FluentValidation）
    ├── PatientValidator.cs
    └── ...
```

**职责边界**:
- ✅ 编排多个领域服务完成业务流程
- ✅ 处理事务边界
- ✅ DTO映射（实体 ↔ DTO）
- ✅ 输入验证
- ❌ 不包含核心业务逻辑（由领域层处理）
- ❌ 不直接访问数据库

**规范**:
- 应用服务以`AppService`后缀命名
- 每个方法对应一个业务用例
- 使用`IDisposable`或依赖注入管理资源
- 所有写操作应包裹在事务中

---

### 3. 领域层 (Domain Layer) - `TcmAiDiagnosis.Domain`

**职责**: 包含核心业务逻辑和领域规则

**包含内容**:
```
TcmAiDiagnosis.Domain/
├── Entities/                   # 领域实体（聚合根、实体、值对象）
│   ├── Patient.cs
│   ├── Visit.cs
│   └── ...
├── Services/                   # 领域服务
│   ├── PatientDomain.cs
│   ├── VisitDomain.cs
│   └── ...
├── Events/                     # 领域事件
│   ├── PatientCreatedEvent.cs
│   └── ...
├── Exceptions/                 # 领域异常
│   ├── DomainException.cs
│   └── ...
├── Constants/                  # 业务常量
│   ├── PermissionConstants.cs
│   └── ...
├── Interfaces/                 # 仓储接口（Repository Pattern）
│   ├── IPatientRepository.cs
│   ├── IVisitRepository.cs
│   └── ...
└── Specifications/             # 规约模式（业务规则）
    ├── PatientSpecification.cs
    └── ...
```

**职责边界**:
- ✅ 核心业务逻辑和规则
- ✅ 领域实体的生命周期管理
- ✅ 领域事件发布
- ✅ 定义仓储接口（不实现）
- ❌ 不依赖基础设施层
- ❌ 不处理技术细节（如数据库、缓存）

**规范**:
- 领域服务以`Domain`后缀命名（当前约定）
- 领域异常继承自`DomainException`
- 仓储接口定义在领域层，实现在基础设施层
- 领域实体应包含业务方法，而不是贫血模型

**职责边界**:
- ✅ 实现仓储接口
- ✅ 数据库访问和迁移
- ✅ 外部API调用

- ❌ 不包含业务逻辑
- ❌ 不依赖应用层或表现层

**规范**:
- 仓储实现以`Repository`后缀命名
- 使用依赖注入注册服务
- 所有外部调用应有异常处理和重试机制
- 数据库操作应支持事务

## 依赖关系规则

### 正确的依赖方向

```
表现层 → 应用层 → 领域层 ← 基础设施层
```

### 禁止的依赖

- ❌ 领域层 → 基础设施层（依赖倒置原则）
- ❌ 基础设施层 → 应用层
- ❌ 表现层 → 领域层（应通过应用层）
- ❌ 应用层 → 表现层

### 依赖注入注册

在`Program.cs`中按层注册服务：

```csharp
// 基础设施层
builder.Services.AddScoped<IPatientRepository, PatientRepository>();
builder.Services.AddDbContext<TcmAiDiagnosisContext>();

// 领域层
builder.Services.AddScoped<PatientDomain>();

// 应用层
builder.Services.AddScoped<IPatientAppService, PatientAppService>();

// 表现层（框架自动注册）
```

## 企业级SSR规范

### 1. 视图模型规范

每个页面必须有独立的ViewModel：

```csharp
// Pages/Doctor/Patients/Index.cshtml.cs
public class IndexModel : PageModel
{
    private readonly IPatientAppService _patientService;
    
    // ViewModel属性
    public PatientListViewModel ViewModel { get; set; }
    
    // GET请求
    public async Task<IActionResult> OnGetAsync(int page = 1)
    {
        ViewModel = await _patientService.GetPatientListAsync(page);
        return Page();
    }
    
    // POST请求
    public async Task<IActionResult> OnPostAsync(CreatePatientInput input)
    {
        if (!ModelState.IsValid)
        {
            ViewModel = await _patientService.GetCreateFormAsync();
            return Page();
        }
        
        await _patientService.CreatePatientAsync(input);
        return RedirectToPage("./Index");
    }
}
```

### 2. API控制器规范

```csharp
[ApiController]
[Route("api/[controller]")]
[Authorize]
public class PatientsController : ControllerBase
{
    private readonly IPatientAppService _patientService;
    
    [HttpGet]
    [RequirePermission("Patient_View")]
    public async Task<ActionResult<PatientListDto>> GetPatients([FromQuery] GetPatientsInput input)
    {
        var result = await _patientService.GetPatientListAsync(input);
        return Ok(result);
    }
    
    [HttpPost]
    [RequirePermission("Patient_Create")]
    public async Task<ActionResult<PatientDto>> CreatePatient([FromBody] CreatePatientInput input)
    {
        var result = await _patientService.CreatePatientAsync(input);
        return CreatedAtAction(nameof(GetPatient), new { id = result.Id }, result);
    }
}
```

### 3. 中间件规范

```csharp
// Middleware/GlobalExceptionMiddleware.cs
public class GlobalExceptionMiddleware
{
    private readonly RequestDelegate _next;
    private readonly ILogger<GlobalExceptionMiddleware> _logger;
    
    public async Task InvokeAsync(HttpContext context)
    {
        try
        {
            await _next(context);
        }
        catch (DomainException ex)
        {
            _logger.LogWarning(ex, "业务异常");
            context.Response.StatusCode = 400;
            await context.Response.WriteAsJsonAsync(new { error = ex.Message });
        }
        catch (Exception ex)
        {
            _logger.LogError(ex, "系统异常");
            context.Response.StatusCode = 500;
            await context.Response.WriteAsJsonAsync(new { error = "系统错误" });
        }
    }
}
```

## 迁移计划

### 阶段一：创建基础设施层（1周）

1. 创建`TcmAiDiagnosis.Infrastructure`项目
2. 移动`EFContext`内容到`Infrastructure/Persistence`
3. 创建仓储接口（Domain层）和实现（Infrastructure层）
4. 更新依赖注入配置

### 阶段二：创建应用层（1周）

1. 创建`TcmAiDiagnosis.Application`项目
2. 移动`Dtos`内容到`Application/DTOs`
3. 创建应用服务和接口
4. 配置AutoMapper映射

### 阶段三：重组领域层（1周）

1. 将`Entities`内容移入`Domain/Entities`
2. 创建领域事件和规约
3. 完善领域服务

### 阶段四：更新表现层（1周）

1. 创建ViewModels目录
2. 更新所有Razor Pages使用ViewModels
3. 更新Controller使用应用服务
4. 添加全局异常处理中间件
